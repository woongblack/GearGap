### 2. 외부 API 전수 조사

#### bloodmallet

- GitHub repo (`bloodytools`, `bloodmallet_web_frontend`) 확인 → 공개 JSON API 없는 줄 알았음
- bloodmallet 페이지 Network 탭 직접 확인 → **공개 API 엔드포인트 발견**

```
GET https://bloodmallet.com/chart/get/trinkets/castingpatchwerk/warlock/destruction

응답:
{
  "item_ids": { "아이템명": 249343 },
  "data": { "아이템명": { "285": 105058, "298": 106095 } },  // ilvl별 DPS
  "data_sources": { "아이템명": "Raid" },
  "sorted_data_keys": [...],  // 성능순 정렬
  "metadata": { 시뮬 조건 }
}
```

URL 패턴: `/chart/get/{type}/{fight_style}/{class}/{spec}`

#### Raidbots 정적 JSON

bloodmallet Network 탭에서 Raidbots 정적 파일 발견 (bloodmallet이 내부적으로 사용):

```
https://www.raidbots.com/static/data/live/instances.json   → 레이드/던전 + 넴드 목록
https://www.raidbots.com/static/data/live/seasons.json     → 시즌 설정
https://www.raidbots.com/static/data/live/bonuses.json     → 아이템 보너스 ID
https://www.raidbots.com/static/data/live/equippable-items.json  → 아이템 메타 (10MB+)
```

모두 공개, 인증 없음. 단, `instances.json`에 보스별 드롭 아이템 정보는 없음.

#### Wowhead tooltip API

bloodmallet Network 탭에서 추가 발견:

```
GET https://nether.wowhead.com/tooltip/item/249343?ilvl=298&dataEnv=1&locale=1

응답: 아이템 이름(한국어), 슬롯, 드롭처(보스명), 난이도, 스탯, 가격
```

드롭처(어느 보스에서 드롭)를 직접 줌. 공개, 인증 없음.

#### Raider.IO API

Go 패키지 문서 확인:
- `GetCharacter()` → 개별 캐릭터 장비 18슬롯 (item_id, item_level, name, gems, bonuses)
- 공개 API, 인증 없음
- 단, 스펙별 상위 N명 집계 엔드포인트는 없음 → 직접 집계 필요

#### Murlok.io

```
...
```
상위 50명 기준, 슬롯별 count. Blizzard API 직접 집계 방식 사용 중.
