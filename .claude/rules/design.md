# GearGap 디자인 / 철학 규칙

## 정체성

GearGap은 "의사결정 대시보드"이지 "정답 자판기"가 아니다.
AskMrRobot(AMR) 실패 패턴을 반복하지 않는 것이 이 프로젝트의 존재 이유.

## 사용 금지 단어 (코드 + UI + 문구 전체)

| 금지 | 이유 | 대안 |
|------|------|------|
| "추천" | 정답이 있다는 인상 | "참고", "후보" |
| "최적" | AMR 패턴 | "효율 점수 기준" |
| "정답" | 블랙박스 인상 | "계산 결과" |
| "1위/2위/3위" | 순위 = 번아웃 가속 | 정렬은 허용, 순위 강조 없음 |
| "AI가 알아서" | 블랙박스 | 계산식 항상 명시 |
| "최고의 아이템" | AMR 패턴 | "이 조건에서 효율 점수가 높은 아이템" |

## 투명성 원칙 — 모든 숫자에 출처

UI에 숫자를 표시할 때 반드시:
- 데이터 출처 표시 (예: `bloodmallet`, `Wowhead`, `Raider.IO`)
- 시뮬 조건 표시 (예: `패치워크 기준 ⚠️`)
- 근거 패널 (?) 아이콘으로 접근 가능

## 가중치 / 슬라이더

- 가중치 하드코딩 절대 금지
- 기본값은 제공하되 유저가 0.5~2.0 범위에서 조정 가능
- 슬라이더 조정 시 효율 점수 즉시 재계산 (1초 이내)

## 근거 패널 (Why this number?)

효율 점수 옆 `(?)` 클릭 시 표시해야 할 정보:
- DPS Gain: 출처 / 시뮬 날짜 / 조건 / 한계
- Time: 출처 / 표본 크기 / 한계
- Drop: 출처 / 표본 크기 / 한계
- 계산식: `(dps_gain × W_dps) ÷ (avg_clear_minutes × W_time) × (drop_rate × W_prob)`

## 시각 디자인

- 다크 배경 + 골드/앰버 포인트 컬러 (WoW 톤)
- 정보 밀도 높지만 깔끔하게
- 모든 숫자에 출처 마이크로카피
- 순위 강조 없음

## Phase별 기능 범위 준수

- Phase 1: 정형 데이터(DPS/시간/확률) 대시보드만
- Phase 2: RAG는 별도 패널(F11)로 분리, 본체와 섞지 않음
- "포트폴리오 멋있어 보이려고" 복잡도 추가 금지
