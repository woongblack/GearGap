# GearGap 디자인 / 철학 규칙

## 정체성

GearGap은 "장비 갭 + 드롭처 로드맵 도구"이지 "정답 자판기"가 아니다.
핵심 한 줄: "내 캐릭터에 뭐가 빠졌고, 어디서 드롭되나"
AskMrRobot(AMR) 실패 패턴을 반복하지 않는 것이 이 프로젝트의 존재 이유.

## 사용 금지 단어 (코드 + UI + 문구 전체)

| 금지 | 이유 | 대안 |
|------|------|------|
| "추천" | 정답이 있다는 인상 | "참고", "후보" |
| "최적" | AMR 패턴 | "착용률 기준", "BiS 후보" |
| "정답" | 블랙박스 인상 | "데이터 기준" |
| "1위/2위/3위" | 순위 = 번아웃 가속 | 정렬은 허용, 순위 강조 없음 |
| "AI가 알아서" | 블랙박스 | 데이터 출처 항상 명시 |
| "최고의 아이템" | AMR 패턴 | "착용률이 높은 BiS 후보" |

## 투명성 원칙 — 모든 숫자에 출처

UI에 숫자를 표시할 때 반드시:
- 데이터 출처 표시 (예: `Murlok.io`, `Wowhead`, `Blizzard API`)
- 샘플 크기 표시 (예: `상위 50명 기준`)
- 갱신 시점 표시 (`scraped_at` 기준)

## 시각 디자인

- 다크 배경 + 골드/앰버 포인트 컬러 (WoW 톤)
- 정보 밀도 높지만 깔끔하게
- 모든 숫자에 출처 마이크로카피
- 순위 강조 없음

## Phase별 기능 범위 준수

- Phase 1 (MVP, 현재): 장비 갭 대시보드만 — 슬롯별 갭 + 드롭처 + 착용률. DPS 시뮬 스코프 외.
- Phase 2 (예정): PRD v0.6 참조 (세부는 Phase 1 완료 후 재정의)
- Phase 3 (장기): PRD v0.6 참조 (세부는 Phase 2 완료 후 재정의)
- "포트폴리오 멋있어 보이려고" 복잡도 추가 금지
