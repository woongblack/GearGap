# PRD: WoW 아이템 추천 시스템
**Product Requirements Document**
작성일: 2026-05-01 | 작성자: 정재웅 | 버전: v0.1

---

## 1. 배경 및 문제 정의

### 배경
현재 World of Warcraft 유저들은 캐릭터 강화를 위해 다양한 사이트를 참고한다.

- **RAGON**: 쐐기 로그 분석, 상위 유저 2차 스텟 통계 제공 → "현재 상태" 분석에 집중
- **Icy Veins / Wowhead**: 스펙별 이론치 스텟 우선순위 가이드 제공 → 정적인 정보

### 문제
> "내 현재 템셋 기반으로 지금 당장 어디 가서 뭘 먹어야 하는지 알 수 없다"

기존 사이트들은 **"현재 상태 분석"에서 끝난다.** "그러니까 다음에 뭘 해야 하는지"를 알려주는 서비스가 없다.

- RAGON: 현재 스텟 분포 보여줌 → So what?
- Icy Veins: 이상적인 스텟 우선순위 알려줌 → But 내 캐릭터 기준이 아님
- Warcraftlogs: 로그 분석 → 플레이 퍼포먼스 중심

**"내 현재 템셋 → 스텟 갭 분석 → 최적 콘텐츠 + 아이템 추천"** 까지 연결하는 서비스가 없다.

---

## 2. 목표

### 핵심 목표
내 캐릭터의 현재 장비를 분석하고, **상위 유저 실제 데이터 기반**으로 스텟 갭을 계산하여, 어느 던전/레이드에서 어떤 아이템을 획득해야 하는지 추천한다.

### 차별점
| 구분 | RAGON | 본 서비스 |
|---|---|---|
| 분석 기준 | WCL 데이터 기반 백분위 | 상위 1% 유저 실제 템셋 기반 |
| 결과물 | 현재 상태 분석 | 다음 행동 추천 |
| 콘텐츠 연결 | 없음 | 레이드 넴드별 / 쐐기 던전별 드롭 매핑 |

---

## 3. 타겟 유저

- 현재 시즌(한밤의 요새 / TWW 시즌 2)을 플레이 중인 WoW 유저
- 아이템 업그레이드 방향을 스스로 판단하기 어려운 중급 유저
- 효율적인 파밍 루트를 찾는 유저

---

## 4. 스코프 (MVP)

### 콘텐츠 범위
- **레이드**: 한밤의 요새 (넴드 8개)
- **쐐기 (미식)**: TWW 시즌 2 던전 풀 (8개 던전)

### 지원 직업/스펙
- MVP: **흑마법사 3개 스펙** (파멸 / 고통 / 악마소환)
- 이후 확장 예정

### 지원하지 않는 것 (MVP 외)
- 힐러 / 탱커 스펙
- 다른 확장팩 콘텐츠
- 공대 구성 추천
- PvP

## 5. 핵심 도메인 개념

### 스텟 갭 (Stat Gap)
> "상위 유저 실제 데이터와 내 캐릭터의 2차 스텟 분포 차이"

- **기준점**: Warcraft Logs 상위 1% 유저들의 실제 착용 아이템 스텟 분포
- **이론치 가이드(Icy Veins)가 아닌, 실제 유저 데이터 기반**으로 산출
- 스펙별로 스텟 우선순위가 다름

| 스펙 | 주요 2차 스텟 우선순위 (초기값, 실데이터로 보정 예정) |
|---|---|
| 파멸 (Destruction) | 신속 > 치명타 > 숙련 > 유연성 |
| 고통 (Affliction) | 숙련 > 신속 > 치명타 > 유연성 |
| 악마소환 (Demonology) | 숙련 > 신속 > 치명타 > 유연성 |

> ※ 스텟 우선순위는 Warcraft Logs 실데이터 수집 후 보정 예정

### 추천 우선순위 로직
```
1. 내 현재 스텟 분포 수집 (Blizzard API)
2. 상위 유저 스텟 기준점 수집 (Warcraft Logs API)
3. 스펙별 우선순위에 따라 갭이 큰 슬롯 탐지
4. 해당 슬롯에 해당하는 아이템을 드롭 DB에서 조회
5. 콘텐츠(레이드 넴드 / 쐐기 던전)와 함께 추천 반환
```

## 6. 기능 요구사항

### 6-1. 캐릭터 조회
- 캐릭터명 + 서버 입력으로 캐릭터 정보 조회
- Blizzard API로 현재 장비 슬롯별 아이템 및 스텟 수집
- 스펙 자동 감지 (파멸 / 고통 / 악마소환)

### 6-2. 스텟 갭 분석
- 스펙별 이상치(상위 1% 평균)와 현재 스텟 비교
- 갭이 큰 슬롯 순으로 우선순위 산정
- 시각적 스텟 분포 차트 제공

### 6-3. 아이템 추천 - 레이드
- 갭 분석 결과를 기반으로 한밤의 요새 넴드별 드롭 아이템 매핑
- 넴드 난이도(일반 / 영웅 / 신화)별 아이템 레벨 반영
- "이 넴드에서 이 아이템을 드세요" 형태로 추천

### 6-4. 아이템 추천 - 쐐기
- 갭 분석 결과를 기반으로 던전별 드롭 아이템 매핑
- 키 레벨에 따른 아이템 레벨 반영
- "이 던전 이 키 레벨 돌면 이 아이템 나와요" 형태로 추천

### 6-5. AI 추천 설명
- Claude API를 활용하여 추천 근거를 자연어로 설명
- 단순 목록이 아닌 "왜 이걸 먹어야 하는지" 설명 포함

## 7. 기술 스택

| 레이어 | 기술 | 선택 이유 |
|---|---|---|
| Backend | Python / FastAPI | AI 연동 친화적, 데이터 수집/가공 생태계 풍부 |
| Database | MySQL | 아이템/드롭 데이터 정형화에 적합 |
| AI | Anthropic Claude API | 추천 근거 자연어 생성 |
| Frontend | React | 심플한 결과 UI, 러닝커브 최소화 |
| 외부 API | Blizzard API, Warcraft Logs API | 캐릭터 데이터 및 상위 유저 스텟 수집 |

---

## 8. 데이터 소스

| 데이터 | 소스 | 용도 |
|---|---|---|
| 캐릭터 현재 장비/스텟 | Blizzard API | 현재 상태 파악 |
| 상위 유저 스텟 분포 | Warcraft Logs API | 스텟 갭 기준점 |
| 넴드별 드롭 아이템 | Blizzard API + 직접 수집 | 레이드 추천 |
| 던전별 드롭 아이템 | Blizzard API + 직접 수집 | 쐐기 추천 |

---

## 9. 시스템 흐름

```
[유저] 캐릭터명 + 서버 입력
    ↓
[Blizzard API] 현재 장비 + 스텟 수집
    ↓
[스펙 감지] 파멸 / 고통 / 악마소환 자동 감지
    ↓
[Warcraft Logs API] 상위 1% 스텟 기준점 조회
    ↓
[스텟 갭 엔진] 슬롯별 갭 계산 + 우선순위 산정
    ↓
[드롭 DB 조회] 갭이 큰 슬롯 → 해당 아이템 탐색
    ↓
[AI 추천 엔진] 레이드 넴드 / 쐐기 던전 추천 + 설명 생성
    ↓
[결과 반환] "○○ 넴드에서 ○○ 아이템 드세요"
```

## 10. 개발 단계

### Phase 1 (MVP)
- [ ] Blizzard API 연동 및 캐릭터 장비 수집
- [ ] Warcraft Logs API 연동 및 상위 유저 스텟 수집
- [ ] 흑마법사 3스펙 스텟 갭 계산 로직
- [ ] 한밤의 요새 + TWW 시즌2 쐐기 드롭 DB 구축
- [ ] 기본 추천 API 개발
- [ ] Next.js 기본 UI

### Phase 2
- [ ] Claude API 연동 - 추천 근거 자연어 설명
- [ ] 스텟 분포 시각화 차트
- [ ] 다른 직업/스펙 확장
- [ ] 아이템 레벨 필터링 (현재 아이레 기준 업그레이드 가능한 것만 추천)

### Phase 3
- [ ] 실데이터 기반 스텟 우선순위 자동 보정
- [ ] 파밍 효율 계산 (드롭 확률 × 스텟 향상도)
- [ ] 주간 추천 루트 제공
