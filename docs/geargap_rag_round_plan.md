# GearGap RAG 라운드 — 세션 9~13 계획

> MVP (세션 1~8) 완료 후 이어지는 RAG/AI 학습 라운드.
> 이 계획은 MVP 끝나봐야 정확해지는 부분이 많음. 지금은 골격만.
> 세션 8 끝나고 다시 보면서 디테일 조정.

## 🎯 RAG 라운드 목표

**기능적 목표:** 사용자가 "이 트링켓 왜 좋은가요?", "흑마 어둠 키 14 갈 만한 빌드 알려주세요" 같은 자연어 질문하면 출처 명시된 답변 제공.

**학습 목표 (개인):**
- 임베딩 모델 사용 경험
- 벡터 DB 운영 (Postgres+pgvector? 또는 별도?)
- RAG 파이프라인 설계 (chunking, retrieval, reranking)
- LLM API 통합 (Claude API / OpenAI / Anthropic)
- 할루시네이션 방지 + 출처 추적

**제약:**
- GearGap 핵심 가치 = 데이터 투명성 → RAG 답변에 출처 무조건 표시
- 할루시네이션 절대 못 일어나도록 (사용자 신뢰)

## 세션 9 — RAG 데이터 소스 결정 + PoC

### 목표
"어떤 데이터를 임베딩할 건가" 결정하고 1개 소스로 PoC.

### 작업
1. **데이터 소스 후보 비교**
   - Wowhead 가이드 (라이선스 확인 필요)
   - Reddit /r/CompetitiveWoW 게시글
   - Wowhead 댓글
   - Discord 클래스 디스코드 발췌 (저작권 회색)
   - YouTube 영상 자막
   - 위키 (Warcraft Wiki)
   
2. **각 소스 평가**
   - 라이선스 / ToS
   - 데이터 품질 / 신뢰도
   - 가공 필요량
   - 신선도 (얼마나 자주 업데이트?)

3. **MVP 1순위 소스 1개 선택 + PoC**
   - 100~500개 문서 수집
   - 청크 분할 (chunking 전략)
   - 임베딩 생성
   - 간단한 코사인 유사도 검색 PoC

### 결정 포인트
- 임베딩 모델: OpenAI / Voyage / 로컬?
- 벡터 저장: Postgres pgvector / Pinecone / Weaviate / Chroma?
- chunking 전략: 고정 길이 / 의미 단위?

### 끝나는 신호
"트링켓 X 어때?" 질문 → 관련 문서 3개 검색 결과 (LLM 답변 X, 검색까지만)

## 세션 10 — 벡터 DB 인프라 + ingestion

### 목표
RAG 데이터 파이프라인 정식 구축.

### 작업
1. **벡터 DB 셋업**
   - Supabase pgvector 활성화 (또는 다른 선택)
   - 스키마 설계
     * documents (원본)
     * chunks (분할 + 임베딩)
     * metadata (출처, 신선도)

2. **Ingestion 파이프라인**
   - 소스별 fetcher
   - 청크 분할 + 임베딩
   - 중복 제거 / 갱신 로직
   - 메타데이터 (출처 URL, 작성일, 신뢰도 점수)

3. **재실행 가능한 구조**
   - 신규 데이터만 처리
   - 임베딩 모델 바뀌면 재생성

### 결정 포인트
- 청크 크기 / 오버랩
- 메타데이터 필터링 (예: 6개월 이상 된 글 제외)
- 임베딩 비용 관리

### 끝나는 신호
~1000~5000 청크 적재됨, 메타데이터 검증됨.

## 세션 11 — Retrieval + Reranking

### 목표
검색 품질 끌어올리기.

### 작업
1. **기본 retrieval**
   - 쿼리 → 임베딩 → top-K 검색
   - 사용자 캐릭터 컨텍스트 포함 (스펙, 컨텐츠 종류)

2. **Reranking**
   - 1차 검색 결과를 LLM이나 reranker 모델로 재정렬
   - cross-encoder 모델 사용 (Cohere / 자체)

3. **하이브리드 검색** (선택)
   - 벡터 + 키워드 (BM25)
   - 아이템 이름 같은 정확 매칭에 유리

4. **평가 셋 구축**
   - 질문 20~50개
   - 각 질문의 기대 결과
   - precision/recall 측정

### 결정 포인트
- Reranker: Cohere API / 로컬 / LLM?
- 하이브리드 vs 순수 벡터
- 평가 메트릭

### 끝나는 신호
20개 평가 질문 중 80% 이상에서 관련 결과 top-3 포함.

## 세션 12 — LLM 통합 + 답변 생성

### 목표
검색된 문서 기반으로 LLM이 답변 생성.

### 작업
1. **LLM 통합**
   - Claude API (Anthropic) 추천 — Anthropic 만든 도구니까 친숙
   - 또는 OpenAI / 로컬 (Ollama)

2. **프롬프트 엔지니어링**
   - 시스템 프롬프트: "출처 인용 필수, 추측 금지"
   - few-shot 예시
   - 사용자 캐릭터 컨텍스트 주입

3. **할루시네이션 방지**
   - 검색 결과에 없으면 "답변 불가" 명시
   - 출처 URL 응답에 포함
   - 신뢰도 점수 표시

4. **답변 형식 표준화**
   - 마크다운 응답
   - 출처 패널 (Prov 컴포넌트 재활용)
   - 갱신 시점

### 결정 포인트
- LLM 선택 + 비용 관리
- 스트리밍 vs 한 번에
- 컨텍스트 윈도우 관리

### 끝나는 신호
실제 질문 → LLM 답변 + 출처 3개 동시 제공.

## 세션 13 — UI 통합 + 검증

### 목표
프론트엔드에서 자연어 질문 → 답변 받기.

### 작업
1. **UI 컴포넌트**
   - 채팅창 (기존 로드맵 화면 옆 or 별도 페이지)
   - 스트리밍 응답 렌더링
   - 출처 카드 (Prov 재활용)
   - 추천 질문 (예: "내 트링켓 어떤가요?")

2. **컨텍스트 자동 주입**
   - 현재 보고 있는 캐릭터 정보 자동 전달
   - "내 ~" 같은 표현 처리

3. **에러 처리**
   - LLM 실패 / 검색 결과 0개 / 토큰 한도

4. **본인 사용 검증**
   - 실제 질문 20개 시도
   - 답변 품질 평가
   - 할루시네이션 발생 케이스 잡기

5. **배포 + Phase 2 완료**

### 결정 포인트
- Rate limiting (LLM 비용)
- 사용자 인증 (남용 방지)
- 피드백 수집 (👍/👎)

### 끝나는 신호
**"GearGap 가서 캐릭터 검색하고 트링켓 질문해봐"** 라고 친구한테 자랑할 수 있음.

## 🤔 RAG 라운드 시작 전 미리 알아두면 좋은 것

### MVP 끝나봐야 명확해지는 것들
- 실제 사용자 (본인) 가 어떤 질문을 하는지
- 어떤 데이터 소스가 가장 가치 있는지
- UI 어디에 RAG 박는 게 자연스러운지

→ 그래서 세션 9 시작 전에 **"본인이 MVP 쓰면서 받은 질문" 메모**를 모아두면 좋아요. 그게 RAG 평가셋의 원천.

### 비용 관리
- 임베딩: OpenAI text-embedding-3-small 기준 $0.02 / 1M tokens
- LLM: Claude Sonnet 기준 $3 / 1M input tokens
- **MVP 단계는 본인만 사용 = 거의 무료**
- 공개 후 트래픽 늘면 비용 관리 필요

### 학습 리소스
- Anthropic Cookbook (RAG 예제)
- LangChain / LlamaIndex (선택)
- pgvector 공식 문서
- Cohere Reranker / Voyage AI

### 위험 신호
- "GearGap이 RAG 챗봇이 되어버림" — 로드맵 도구 정체성 흐려짐
- 할루시네이션 1번 = 신뢰 크게 깨짐
- 데이터 소스 라이선스 문제

→ RAG는 **메인 기능이 아니라 보조 기능**으로 두는 게 핵심.

## 📊 RAG 라운드 끝나면 진척도

```
세션 8 끝: MVP 100% 완료
세션 13 끝: MVP + RAG (Phase 2 일부) 완료
```

이 시점에 GearGap이 가진 것:
- ✅ 로드맵 도구 (사용자 가치)
- ✅ RAG 챗봇 (학습 가치 + 차별화)
- ✅ 데이터 인프라 (Murlok + Raidbots + RAG corpus)
- ✅ 포트폴리오 자료 ("end-to-end RAG 시스템 구현")

```

□ 평가셋에서 회귀 테스트
□ 출처 표시 정상 작동
□ 데이터 소스 ToS 변경 없는지 (가끔)
```
